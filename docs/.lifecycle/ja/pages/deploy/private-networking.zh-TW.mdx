# サービス内ネットワーク

サービス内ネットワーク（プライベートネットワーキング）は、サービスが固定のホスト名を使用して、同じプロジェクトに属するネットワークに接続できるようにします。

## ホスト名の確認

「Networking」セクションを開き、「Private」のセクションでホスト名を確認できます。

![ホスト名を見つける](/deploy/private-networking/find-hostname.png)

## サービスへの接続

他のサービス（この例では PostgreSQL）がこのサービスに接続する場合は、ホスト名を使用して接続します。また、[PostgreSQLのドキュメントによると](/marketplace/postgresql)、そのポートは `POSTGRES_PORT` 環境変数に注入されるため、接続情報は次のように入力します。

- ホスト名：前述の手順で確認したホスト名。この例では `postgresql.zeabur.internal`です。
- ポート：`POSTGRES_PORT` 環境変数から値を取得し、[デフォルトは5432です](https://github.com/zeabur/zeabur/blob/bd565389cf1868a917333d0f74003b4aad6ef0ce/prebuilts/postgresql.toml#L14)。

## 注意事項

- サービス名を変更しても、**ホスト名は変更されません**。
