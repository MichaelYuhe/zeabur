# 監視パス

サービスの展開が完了すると、コードを頻繁に更新することがあるかもしれませんが、すべてのコードの更新に再構築を必要とするわけではありません。

たとえば、フロントエンドとバックエンドが一体化したアプリケーションでは、バックエンドの一部のコードを更新しただけで、フロントエンドのサービスも再構築されることがあります。このようなことを避けたい場合は、監視パスを設定で変更するだけです。

### 第一ステップ：サービス設定ページを開く

変更を加えたいサービスをクリックし、設定ページを展開し、下にスクロールして、**監視パス** 項目を見つけてサービスの監視パスを変更します。

![setting](/deploy/watch-path/setting.zh-CN.png)

### 第二ステップ：監視パスを変更する

デフォルトの監視パスは `*` です。これはリポジトリ全体を監視し、リポジトリ内の任意のファイルの更新がサービスの再構築をもたらします。

`.gitignore` のファイル形式に従って独自の監視パスを変更できます。

唯一の違いは、`.gitignore` が無視を表すのに対し、ここでは監視を表します。

たとえば、フロントエンドとバックエンドが一体化したアプリケーションを展開し、プロジェクト構造が次のようになっているとします。

![repo](/deploy/watch-path/repo.png)

このリポジトリには2つのサービスが展開されており、1つのサービスのルートディレクトリが `client` に、もう1つのサービスのルートディレクトリが `server` に設定されていますが、これは構築のパスを指定しているだけであり、さらなる隔離を行うには、**監視パス** を構成する必要があります。

たとえば、クライアントサービスの **監視パス** を `/client` に設定すると、`server` フォルダーで行った更新は自動的にクライアントサービスの再構築をトリガーしません。
