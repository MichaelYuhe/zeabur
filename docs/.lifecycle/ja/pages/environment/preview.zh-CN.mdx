# プレビュー環境

時々、プルリクエストにプレビュー環境をデプロイしたい場合があります。変更を本番ブランチにマージする前に変更をテストするのに役立ちます。
私たちはZeaburで提供されているすべてのサービスに対して、これを実現するための簡単な方法を提供しています。

このドキュメントでは、Zeaburでのプルリクエストのプレビュー環境を簡単に有効にする方法を示します。

<Callout>この機能はDeveloperプランとTeamプランのユーザーにのみ公開されています。</Callout>

## Gitサービスを選択する

[はじめ方](/get-started) で説明したように、ZeaburにはGitサービスとMarketplaceサービスの2種類のサービスがあります。
プルリクエストのプレビュー環境はGitサービスにのみ適用されます。

Gitサービスをまだ作成していない場合は、[はじめ方](/get-started) の指示に従って作成できます。

## プルリクエストのプレビュー環境を有効にする

プレビュー環境を有効にしたいサービスを開き、画面下部にスクロールして設定タブを展開します。
**プルリクエストのプレビュー**という名前のスイッチボタンが表示されます。スイッチボタンをクリックしてプルリクエストのプレビュー環境を有効にします。

![enable](/environment/preview/enable.png)

## プレビューを確認する

プルリクエストのプレビュー環境を正常に有効にした後、Gitリポジトリでプルリクエストを作成することができます。
プルリクエストを作成すると、Zeaburは自動的にプレビュー環境を作成します。

また、Zeaburは自動的に一意のドメインを割り当てるため、プレビューサービスに簡単にアクセスできます。
このドメインは当社のコンソールで確認できますが、Gitリポジトリでデプロイリンクを直接クリックすることもできます。

![link](/environment/preview/link.png)

心配しないでください。プルリクエストがマージされるか、閉じられると、プレビュー環境は自動的に削除されます。
