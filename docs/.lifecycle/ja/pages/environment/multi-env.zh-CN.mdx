# マルチエンバイロメント

プロジェクトでは、通常、本番環境と開発環境など、複数の環境の隔離が必要です。

Zeabur は、このような機能を提供しています。

## 環境とは

環境とは、サービスが実行される場所です。サーバー、クラスター、クラウド、またはローカルマシンなどがそれに該当します。

異なる環境は互いに独立しています。それらは異なる構成や異なるサービス、さらには異なるバージョンのサービスを持つことができます。

Zeabur は、1つのプロジェクトで複数の環境を作成するのを支援します。

## なぜ複数の環境が必要なのか

1つのプロジェクトでは、サービスを異なる環境にデプロイする必要があるかもしれません。たとえば、本番環境や開発環境にサービスをデプロイする必要があるかもしれません。

本番環境はお客様にサービスを提供するためのものです。安定して信頼性が高い必要があります。本番環境にサービスを頻繁にデプロイしてはいけません。

開発環境は開発者向けのものです。デプロイやテストが容易である必要があります。開発環境にサービスを頻繁にデプロイするべきです。

## 複数の環境を管理する方法

### ステップ1: 新しい環境を作成

プロジェクトに入った後、**新しい環境** ボタンをクリックし、新しいプロジェクト環境を作成するために環境の名前を入力します。

![create](/environment/create-env.zh-CN.png)

作成が完了すると、現在のプロジェクト内のすべてのサービスが自動的にコピーされ、新しい環境で再構築されます。

注意すべき点として、サービス自体に露出されている環境変数以外はコピーされないことです。

### ステップ2: 異なるブランチをバインド

デプロイされたソースコードについて、同じサービスに異なる環境で異なるブランチをバインドすることができます。

サービスをクリックして、下にスクロールして設定ページを開き、**現在のブランチ** 項目でその環境にバインドするブランチを選択します。

たとえば、dev ブランチを選択すると、サービスのステータスが dev ブランチにバインドされ、dev 環境のドメインを別に設定することで、dev 環境のオンラインステータスをリアルタイムで確認することができます。
