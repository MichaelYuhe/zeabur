# プレビュー環境

時折、プルリクエストにプレビュー環境をデプロイしたい場合があります。変更を本番環境にマージする前に、変更をテストするのに便利です。
Zeabur上でデプロイされたすべてのサービスには、この機能を簡単に実現する方法が用意されています。

本ドキュメントでは、Zeabur上でプルリクエストのプレビュー環境を簡単に有効にする方法を説明します。

<Callout>この機能はDeveloperプランとTeamプランのユーザーにのみ開放されています</Callout>

## Gitサービスを選択

[はじめる](/get-started) に記載されているように、ZeaburにはGitサービスとMarketplaceサービスの2種類のサービスがあります。
プルリクエストのプレビュー環境はGitサービスにのみ適用されます。

まだGitサービスを作成していない場合は、[はじめる](/get-started) の指示に従って作成できます。

## プルリクエストのプレビュー環境を有効にする

プレビュー環境を有効にしたいサービスを開き、画面下部にスクロールして設定タブを展開します。
**Pull Requests Preview** という名前のスイッチボタンが表示されます。このスイッチボタンをクリックしてプルリクエストのプレビュー環境を有効にします。

![有効化](/environment/preview/enable.png)

## プレビューを確認する

プルリクエストのプレビュー環境が正常に有効になると、Gitリポジトリでプルリクエストを作成できます。
プルリクエストを作成すると、Zeaburは自動的にプレビュー環境を作成します。

さらに、Zeaburはユニークなドメインを自動的にバインドし、プレビューサービスに簡単にアクセスできるようにします。
このドメインは当社のコントロールパネルで確認できる他、Gitリポジトリでデプロイリンクを直接クリックすることもできます。

![リンク](/environment/preview/link.png)

プルリクエストがマージされるか閉じられると、プレビュー環境は自動的に削除されるため、安心してご利用いただけます。
