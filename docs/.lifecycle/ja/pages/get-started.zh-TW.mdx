# クイックスタート

## ステップ1：アカウントの作成

ゼーバーでサービスをデプロイするには、まずアカウントを作成する必要があります。
ゼーバーは現在、GitHubを経由したログインのみをサポートしています。まだGitHubアカウントをお持ちでない場合は、[こちら](https://github.com/join) から作成できます。

[ログインページ](https://dash.zeabur.com/login) にアクセスし、GitHubでログインするボタンをクリックします。
このボタンをクリックすると、GitHubの認可ページにリダイレクトされます。 "Authorize" ボタンをクリックして、ゼーバーがあなたのGitHubアカウントにアクセスできるようにします。

![sign in](/start/sign-in.png)

## ステップ2：プロジェクトの作成

ゼーバー上で、1つのプロジェクトは実際のプロジェクトを表します。1つのプロジェクトで複数の異なるタイプのサービスをデプロイして、実際のニーズに応えることができます。

認可が完了すると、自動的にゼーバーのダッシュボードページにリダイレクトされます。
**プロジェクトの作成** ボタンをクリックして、新しいプロジェクトを作成します。または、ショートカットキー `CMD + k` を使用して、**プロジェクトを作成** を選択します。

プロジェクトがどのリージョンにあるかを選択するためのポップアップが表示されます。ゼーバーは現在、2つのリージョン、つまり `asia-east` に位置するGCPと `us-west` に位置するAWSのクラスタをサポートしています。
サービスをデプロイするリージョンを選択し、後ほどプロジェクトの設定で名前を変更できるようにします。

![new project](/start/create-project.png)

<Callout>
  プロジェクトを作成するのが初めての場合は、ガイドに従い、サービス規約を読んで同意する必要がある場合があります
</Callout>

## ステップ3：サービスのデプロイ

作成したばかりのプロジェクトをクリックすると、ゼーバーがそのプロジェクトの詳細ページを開きます。

![project](/start/project.png)

サービスをデプロイするには、**サービスを追加** ボタンをクリックするだけです。サービスの種類を選択するためのポップアップが表示されます。

![select service type](/start/select-service-type.png)

ゼーバーは **Git Service** と **Marketplace Service** の2種類のサービスをサポートしています。

- **Git Service** は、あなたのソースコードをデプロイできるようにします。ゼーバーは自動的にあなたのソースコードをビルドし、デプロイします。
  初めて使用する場合は、ゼーバーにGitHubへのアクセス権を設定する必要があります。

- **Marketplace Services** では、一般的なサービスのマーケットプレイスからサービス（例えばデータベースやオープンソースツール）をデプロイできます。当社のオープンソースのサービスのリストは[こちら](https://github.com/zeabur/zeabur/tree/main/marketplace)でご覧いただけます。

デプロイしたいサービスの種類を選択して、デプロイしたいサービスを自由に探索できます。

## 次のステップ

デプロイが成功したら、[ドメインバインディング](/deploy/domain-binding)を学んでサービスにドメインをバインドし、自動的にTLS証明書を設定する方法や、[環境変数](/environment/variables)を学んで、Zeaburがどのようにして変数を自動的にあなたのプログラムに注入するかを理解できます。
