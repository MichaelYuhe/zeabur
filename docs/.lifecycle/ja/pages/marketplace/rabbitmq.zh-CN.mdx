# RabbitMQ サービスのデプロイ

Zeabur は RabbitMQ サービスを提供しており、Zeabur 上でワンクリックで RabbitMQ メッセージキューをデプロイすることができます。

<Callout type="info" emoji="ℹ️">
    現在の RabbitMQ サービスのバージョンは 3-management です。
</Callout>

## RabbitMQ サービスのデプロイ

プロジェクト内で、**新規サービス作成** をクリックし、**プリビルド** をクリックして、ポップアップリストの検索バーで `RabbitMQ` を検索します。

![deploy](/deploy/common/select-service.en-US.png)
デプロイをクリックすると、Zeabur が自動的に RabbitMQ サービスをデプロイしてくれます。

## 環境変数
RabbitMQ サービスをデプロイした後、Zeabur は関連する環境変数を自動的に他のサービスに注入します。

 - `RABBITMQ_HOST`
 - `RABBITMQ_PORT`
 - `RABBITMQ_DEFAULT_USER`
 - `RABBITMQ_DEFAULT_PASS`

時には、環境変数に加えて `DATABASE_URL` を新たに追加することで次のような形式で使用することもできます：

```bash
amqp://<RABBITMQ_DEFAULT_USER>:<RABBITMQ_DEFAULT_PASS>@<RABBITMQ_HOST>:<RABBITMQ_PORT>/<DATABASE_NAME>
```

ここでの `<DATABASE_NAME>` は自分で追加した Vhost 名です。

## オンライン管理インターフェース

RabbitMQ サービスはオンラインの管理インターフェースを提供しており、ブラウザから RabbitMQ サービスを管理することができます。

`http://<RABBITMQ_HOST>:15672` にアクセスすることで RabbitMQ サービスのオンライン管理インターフェースにアクセスできます。

## DSN の設定

`$RABBITMQ_URI` という変数を通じて、RabbitMQ サービスの DSN（つまり RabbitMQ サービスのネットワークアドレス）を使用することができます。
