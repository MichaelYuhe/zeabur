# はじめに

## ステップ1: サインアップ

はじめに、Zeaburで**アカウントを作成**する必要があります。
サインアップするには**GitHubアカウント**が必要です。お持ちでない場合は[こちら](https://github.com/join)で作成できます。

[ログインページ](https://dash.zeabur.com/login)に移動すると、GitHubでサインアップするためのボタンが表示されます。
ボタンをクリックすると、GitHubの認可ページにリダイレクトされます。"Authorize"ボタンをクリックして、ZeaburがGitHubアカウントにアクセスすることを許可します。

![サインイン](/start/sign-in.png)

## ステップ2: 新しいプロジェクトを作成

プロジェクトはサービスのグループとして表示されます。プロジェクトに複数のサービスをデプロイして、簡単にサービス間で通信できます。

認可後、Zeaburダッシュボードに自動的にリダイレクトされます。
**プロジェクトを作成**ボタンをクリックして新しいプロジェクトを作成します。また、クイックアクションメニューを開くには`CMD + k`を入力し、`Create Project`を選択できます。

プロジェクトのリージョンを選択するためのモーダルが表示されます。Zeaburは現在`asia-east`にGCPサーバーと`us-west`にAWSサーバーをサポートしています。
サービスをデプロイするリージョンを選択し、プロジェクト名を無作為に生成されます。後でプロジェクト設定で変更できます。

![新しいプロジェクト](/start/create-project.png)

<コールアウト>
  Zeaburでプロジェクトを作成するのが初めての場合は、続行するために利用規約に同意して読む必要がある場合があります。
</コールアウト>


## ステップ3: サービスをデプロイ

作成したプロジェクトをクリックすると、Zeaburがプロジェクトページを開きます。

![プロジェクト](/start/project.png)

サービスをデプロイするには、**サービスを追加**ボタンをクリックします。モーダルが表示され、サービスの種類を選択できます。

![サービスの種類を選択](/start/select-service-type.png)

Zeaburは**Gitサービス**と**マーケットプレイスサービス**の2種類のサービスをサポートしています。

- **Gitサービス**ではGitHubからソースコードをデプロイできます。Zeaburが自動的にソースコードをビルドしてデプロイします。
  最初にGitHubのアクセスを構成する必要があり、Zeaburがリポジトリにアクセスできるようにします。

- **マーケットプレイスサービス**では、一般的なサービス（データベースやオープンソースツールなど）のマーケットプレイスからサービスをデプロイできます。オープンソースのマーケットプレイスは[こちら](https://github.com/zeabur/zeabur/tree/main/marketplace)で確認できます。

必要なサービスの種類を選択し、デプロイしたいサービスを検索できます。

## 次の手順

デプロイした後は、[ドメインバインディング](/en-US/deploy/domain-binding)について学ぶことで、自動的に構成されたTLS証明書でサービスを公開できるようになります。また、[環境変数](/en-US/environment/variables)について学ぶことで、Zeaburがプログラムに自動的に変数を注入できる方法を理解できます。
